**Decisions reached (US23 design).**
- **Both axes of the scorecard are derived from the records, never hard-coded** — the model list
  and the dimension list come from the keys present in `scores.jsonl` (minus the identity keys
  `model`/`page`/`gold`), so a newly scored model or a new metric appears with no code edit (AC3).
  Implies a future `ocr-report` / OCR-bench module dossier.
- **The per-dimension summarizer is dispatched by the *value kind*, not by a hard-coded
  name→kind map** — pure-int → representative median (count-like); any-float → mean (ratio/score);
  str/bool → dominant value (categorical); empty/all-null/mixed → an explicit gap. This keeps the
  aggregator data-driven while matching the report's buckets (`hyphen_artifacts`/`citation_groups`
  medianed, `dup_pct`/`teds`/`text_edit_distance`/`latency` meaned, `finish_reason`/`cjk_present`
  dominant). Implies the OCR-bench aggregation dossier.
- **A missing cell renders an explicit gap `—`, never a false `0`** (AC4) — a not-applicable or
  unmeasured cell must read as "not measured", not "scored badly".
- **The verdict ranks only dimensions with a *taught* direction** — a small `_HIGHER_IS_BETTER` /
  `_LOWER_IS_BETTER` registry (rule 02 encoded knowledge); a new/neutral dimension shows in the
  table but is unranked until its direction is a one-line addition. A single weighted headline
  ranking is deferred (the story's "Later stages").
- **Re-scored rows collapse last-wins per (model, page, gold-tier)** before aggregating, so the
  append-only `scores.jsonl` does not double-weight a re-scored page — this satisfies the
  expectation the US22 minute handed to the US23 aggregator.
- **Never crash / never call an LLM (rule 02)** — malformed non-JSON lines are skipped (a
  non-pipeline-produced line is not a record; mirrors `score_ocr._manifest_fields`); a well-formed
  record with no string `model` or a non-string `page` is quarantined (`stage: "ocr-report"`); a
  non-UTF-8 file → a clean `BadParameter` (exit 2); non-finite floats → gap.
