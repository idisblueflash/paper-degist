## [2026-07-06] session 7c44bfc0 — US 3 default OCR model set to deepseek-ocr-2; US 4 cancelled

**Intent.** User asked (a) whether US 4 (text → MD) should be cancelled now that
the OCR model converts PDF → MD directly, and (b) whether `deepseek-ocr-2` (the
top-scoring model) should be the default PDF-parse model for US 3.

is **silent** — no compiled claim about which OCR model scored highest or the
output format. Only the four existing dossiers, none touching OCR scoring. Same

**Verified from data (not canon).** Aggregated `scores.jsonl` (857 records = 632
reference-free + 225 gold-accuracy, 3 models):
- **Gold accuracy (OmniDocBench subset):** deepseek-ocr-2 won **both** metrics —
  text edit distance 0.117 (lowest) and table TEDS 0.727 (highest); vs
  deepseek-ocr@8bit (0.157 / 0.717) and qwen3-vl-4b (0.234 / 0.692).
- **Reference-free:** deepseek-ocr-2 clean (hyphen artifacts 0.14) vs the 8-bit
  quant's **12.18** (quantization wrecks hyphenation); 0 non-stop finishes;
  ~19 s/page (qwen ~65 s).
- **Format:** every registered OCR model's prompt is literally *"Convert the
  document to markdown."* (`src/paper_degist/ocr_page.py:175-191`), and the gold
  metrics compare against gold **Markdown** — so the OCR path emits MD directly,
  with no intermediate plain-text stage.
