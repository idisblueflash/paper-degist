### Durable conventions captured (each implies a dossier)

**1. discover candidate JSONL — the common schema.** → implies a module dossier
"discover candidate schema".
- `src/paper_degist/discover.py`, `Candidate.to_record()`. Every discovered paper,
  regardless of source, emits one JSONL record in a **source-agnostic common
  schema**: always-present `title`, `authors` (list), `abstract` (may be `null`),
  `abstract_present` (bool), `url`, `published`, `source`, `source_id`; plus `doi`
  and `tldr` **only when the record carries them**.
- `abstract_present` is emitted deliberately so a *downstream* narrowing step
  (US26) can drop an abstract-less hit cheaply — discovery itself stays wide and
  never drops on missing abstract (US25 AC3: coarse, high-recall by contract).
- Sources are a **registry** keyed by `--source` (rule 02), not per-source
  branches: `arxiv` (Atom, no key) and `s2` (Semantic Scholar JSON, optional key,
  carries `externalIds.DOI` and an optional `tldr` one-line summary). A new source
  is one adapter entry.
- Classify-then-dispatch in two layers: on `--source` (unknown → quarantine
  offline, no network) then on the transport result (hits → emit + a `discover`
  success record with `result_count`; empty → quarantine `empty-result`; HTTP
  error → quarantine `api-error`) — distinct reasons, never crash, never LLM.

**2. embed-text transport + registry.** → implies a module/architecture dossier
"embed-text LM Studio transport".
- `src/paper_degist/embed_text.py`. The **transport is the hard-won invariant**
  carried verbatim from `ocr_page` (US20 report §3): build the JSON body in
  Python, POST with `curl --data @body.json`, **sequentially**, with a recovery
  gap and retry — **never `urllib`** (its empty-body POST 502s and takes the MLX
  worker down), **never concurrent**. Constants: `DEFAULT_ENDPOINT =
  http://localhost:1234/v1/embeddings`, `DEFAULT_GAP = 7.0` s, `DEFAULT_ATTEMPTS
  = 3`.
- Dispatch on the transport result: 200 → save; **4xx → fail fast** (deterministic
  `ClientRequestError`, do not burn the retry budget); 5xx/empty/malformed-200 →
  wait the gap and retry, then quarantine `server unreachable`. A malformed 200
  (truncated JSON, non-list `embedding`) is treated as a flap and retried, never
  crashes.
- Models are a **registry** (`EmbedModelSpec`) of `(query_prefix, doc_prefix)`
  pairs keyed by model id; the `--role` (`query`|`document`) selects the prefix.
  Prefix matters — `nomic-embed-text-v1.5` wants `search_query: ` vs
  `search_document: `, and getting it wrong silently degrades ranking. A new model
  is one registry entry.
- Vectors are **content-addressed** by `sha256(model, role, text)` (all three,
  length-delimited) under `out/embeddings/<model>/<hash>.json`; an existing file is
  an idempotent skip. `embed_text()` returns the path on success or `None` on
  quarantine.

**3. DOI normalization — today vs. deferred.** → implies a module dossier "DOI
normalization / doi_from".
- Extraction exists today: `src/paper_degist/resolve_oa.py::doi_from(text)` reads a
  DOI out of a `doi.org` link, a publisher URL that embeds the DOI in its path, or
  a bare `10.\d+/…` string. This is the **DOI-reading** primitive.
- A **normalized-DOI dedup key** (strip scheme + `doi.org`/`dx.doi.org` prefix,
  lowercase because DOIs are case-insensitive) is specified by **US 14
  (`dedup-inputs`) — NOT yet built** (status `—` in the index). US 26's
  deterministic pass is spec'd to **reuse US 14's normalization**, so building US26
  before US14 means the normalization helper must be written (or extracted) here.
  Open thread below.

**4. US 26 abstract-filter — two-pass, LLM-free.** → implies a module dossier
"abstract-filter (US26)".
- `user-stories/us-26-abstract-filter-embedding.md`. New step `abstract-filter`
  over a candidate JSONL + a `--topic`. Narrows US25's wide net in **two passes,
  no LLM**:
  - **Deterministic first** (pure, offline, free): dedup by normalized DOI (reuse
    US14) and drop `abstract_present: false`; each drop a `filtered` manifest
    record (`stage: "abstract-filter"`) with a reason separating dedup-doi from
    no-abstract — **before any embedding call**.
  - **Embedding second**: embed the `--topic` once as a `query` and each surviving
    abstract as a `document` via `embed-text` (US24), score **cosine similarity**,
    cut below a **threshold measured against a real sample** (rule 06 phase 2, like
    the HTML-density / OCR constants — not guessed).
  - Output: ranked JSONL, descending similarity, score attached — a drop-in to
    `fetch-one`.
  - A per-candidate `embed-text` quarantine (server down mid-run) quarantines only
    that candidate; the batch still completes. The intent-aware LLM judge on the
    ambiguous middle is explicitly a **later** story.
